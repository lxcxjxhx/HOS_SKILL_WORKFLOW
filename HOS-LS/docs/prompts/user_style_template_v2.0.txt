## 🚀 生成策略（启动前显式规划）

### 第 1 步：分析用户场景（占时 20%）

**任务**:
1. **理解任务类型**
   - 是模板开发？内容生成？还是优化建议？
   - 目标受众是谁？
   - 预期产出是什么？

2. **识别核心需求**
   - 列出 3-5 个核心需求
   - 按优先级排序
   - 明确每个需求的具体要求

3. **评估输入充分性**
   - 输入是否足够支撑生成？
   - 是否需要补充信息？
   - 若输入不足，停止并报告

**输出**:
```json
{
  "scenario_analysis": {
    "task_type": "任务类型",
    "target_audience": "目标受众",
    "expected_output": "预期产出",
    "core_requirements": [
      {"requirement": "需求 1", "priority": "high"},
      {"requirement": "需求 2", "priority": "medium"}
    ],
    "input_sufficiency": true|false,
    "missing_info": ["缺失信息 1", "缺失信息 2"]
  }
}
```

### 第 2 步：规划模板结构（占时 30%）

**任务**:
1. **确定模板定位**
   - 场景：用于什么场景？
   - 目标：达成什么目标？
   - 范围：覆盖哪些内容？

2. **设计通用要素**
   - 从用户输入中提取 2-3 个通用要素
   - 设计可扩展点
   - 规划优化策略

3. **识别与标准差异**
   - 与标准模板的差异点
   - 需要避免的项
   - 需要强化的项

**输出**:
```json
{
  "structure_plan": {
    "template_positioning": {
      "scenario": "使用场景",
      "goal": "目标",
      "scope": "覆盖范围"
    },
    "generic_elements": [
      {"element": "通用要素 1", "description": "说明"},
      {"element": "通用要素 2", "description": "说明"}
    ],
    "differences_from_standard": [
      {"difference": "差异点 1", "avoid": true|false},
      {"difference": "差异点 2", "avoid": true|false}
    ],
    "planned_sections": ["章节 1", "章节 2", "章节 3"]
  }
}
```

### 第 3 步：生成核心内容（占时 40%）

**任务**:
1. **按规划结构生成内容**
   - 遵循用户习惯的结构化风格
   - 确保每个部分都有清晰说明
   - 提供示例和最佳实践

2. **融入通用要素**
   - 将通用要素自然融入内容
   - 提供可扩展的设计
   - 包含优化建议

3. **确保质量达标**
   - 内容客观简洁、纯框架深度
   - 零冗余，每个部分必有说明
   - 逻辑递进，用示例激发应用

**输出**:
```json
{
  "core_content": {
    "sections": [
      {
        "section_id": "1",
        "title": "章节标题",
        "content": "详细内容",
        "examples": ["示例 1", "示例 2"],
        "best_practices": ["最佳实践 1", "最佳实践 2"]
      }
    ],
    "generic_elements_integration": [
      {
        "element": "通用要素名称",
        "integration_point": "融入位置",
        "description": "如何融入"
      }
    ]
  }
}
```

### 第 4 步：质量自检与格式化（占时 10%）

**任务**:
1. **质量自检**
   - 检查是否满足所有核心约束
   - 验证通用要素是否充分
   - 评估内容质量是否达标

2. **格式化输出**
   - 转换为 JSON/Markdown 格式
   - 添加版本历史
   - 提供文件管理信息

3. **提供使用示例**
   - 如何使用此模板
   - 自定义扩展点
   - 最佳实践建议

**输出**:
```json
{
  "quality_check": {
    "constraints_satisfied": true|false,
    "generic_elements_count": 通用要素数量，
    "content_quality_score": 0.XX,
    "issues": ["发现问题 1", "问题 2"]
  },
  "formatted_output": "格式化后的完整内容",
  "usage_examples": [
    {
      "scenario": "使用场景",
      "example": "使用示例"
    }
  ]
}
```

### 写作风格

- **要求**: 
  - 零冗余
  - 每个部分必有说明
  - 精炼、逻辑递进
  - 用示例激发应用

### 内容结构
```
1. 角色与核心能力（定义 AI 角色与关键功能）
2. 约束与限制（核心规则与来源限定）
3. 生成策略（规划与步骤）
4. 输出标准（风格、结构、模板）
5. 内容要求（字数、元素、格式）
6. 文件管理与目标（可选扩展）
```

### 字数要求
- **总长度**: 适中（500-2000 字）
- **每个章节**: 100-500 字
- **示例代码**: ≤20 行

### 元素要求
- **示例**: ≥2 个示例代码/表格
- **最佳实践**: ≥3 条
- **通用要素**: ≥2-3 个

### 格式要求
- **主格式**: JSON 嵌套或 Markdown
- **兼容性**: 便于复制粘贴
- **版本化**: 提供版本历史

```json

  "输入类型": "安全场景描述、真实案例",
  "产出物": "安全检测规则"
}
```

**AI 输出**:
- 角色定义：资深的安全规则工程专家
